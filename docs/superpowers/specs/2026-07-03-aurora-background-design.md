# 实时监控台 · 极光光晕背景优化设计

- 日期：2026-07-03
- 状态：设计已确认
- 范围：仅前端 `index.html`，后端不动

## 1. 需求背景

当前背景整体观感僵硬，来源有三：

1. `body` 三层 `linear-gradient` 使用硬色标，形成锐利的斜切色带（左上青绿块、中下琥珀带）
2. `.grid-bg` 56px 网格线铺满全屏，无淡出
3. `.signal-bg` 两条约 1% 宽的细斜线横穿页面

用户诉求（已逐条确认）：

- 硬边色块换成柔和光晕，整体朦胧自然
- 增加克制的慢速动态效果
- 改动范围：背景为主 + 与背景直接相关的协调微调，不动布局与功能

## 2. 设计方案

### 2.1 底色层（body）

移除三层硬色标渐变，换成一层无断点的平滑渐变：

- 浅色模式：`linear-gradient(180deg, #f8fafc 0%, #f0fdfa 50%, #f8fafc 100%)`（雾白 → 极淡薄荷青 → 雾白）
- 深色模式：`linear-gradient(180deg, #020617 0%, #0f172a 52%, #030712 100%)`（深空色平滑过渡）

### 2.2 极光光斑层

在现有背景装饰容器（`.pointer-events-none.fixed.inset-0`）内新增 3 个 `div.aurora-blob`：

| 光斑 | 颜色基调 | 位置 | 尺寸 | 透明度（浅/深） |
| --- | --- | --- | --- | --- |
| blob-1 | 青绿 #2dd4bf 系 | 左上 | 约 55vw | 0.16 / 0.22 |
| blob-2 | 琥珀 #f59e0b | 右侧中部 | 约 45vw | 0.10 / 0.14 |
| blob-3 | 冷蓝 #38bdf8 | 左下 | 约 50vw | 0.12 / 0.16 |

- 使用 `radial-gradient(closest-side, 颜色, transparent)` 自然羽化
- 不使用 `filter: blur()`：径向渐变本身完全平滑，同时避免额外 GPU 开销
- 深色模式下光晕略强（深底上极光效果更明显）

### 2.3 慢速动效

- 每个光斑独立 keyframes：小范围漂移（±5% 左右）+ 缩放呼吸（1 ↔ 1.06）+ 轻微透明度变化
- 周期分别 75s / 90s / 105s，`alternate` 往复，三个光斑节奏错开
- 仅动画 `transform` 与 `opacity`（GPU 合成层，常驻大屏不吃 CPU）
- `@media (prefers-reduced-motion: reduce)` 下停止全部背景动画

### 2.4 网格柔化

- 保留 `.grid-bg`（监控台科技感标识）
- 新增径向遮罩 `mask-image`（含 `-webkit-` 前缀）：页面中上部可见，向四周渐隐
- 网格线透明度 0.07 → 0.05

### 2.5 删除项

- `.signal-bg` 样式及对应 div 整体删除

### 2.6 协调微调

- `.glass` 卡片透明度略降：浅色 0.84 → 约 0.78，深色 0.78 → 约 0.72，让光晕隐约透过玻璃
- 具体数值实施时目测微调，底线是卡片文字可读性不下降

## 3. 非目标

- 不改布局、组件结构、图表配色、交互逻辑
- 不引入任何新依赖（纯 CSS 实现）
- 不改后端 Go 代码

## 4. 验证标准

1. `go run ./cmd/server` 本地运行（无数据库时自动使用演示数据）
2. Playwright 截图核对浅色 / 深色两套主题效果
3. 无横向滚动条；卡片文字可读性不下降
4. 背景动画仅涉及 transform/opacity；reduced-motion 环境下静止
5. `go test ./...` 通过（确认后端无恙）
