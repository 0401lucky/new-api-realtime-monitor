# 实时监控台 UI 精修设计方案

日期:2026-07-03
状态:提案(未整体实施);其中背景部分已由 `2026-07-03-aurora-background-design.md` 细化取代并落地,其余各项待定。
范围:仅 `index.html`(样式层 + 渲染函数中的类名与图表配色),不改任何业务逻辑、接口与数据流。

## 背景与目标

当前 UI 为青绿色玻璃拟态风格,主要问题:

1. 背景的两道斜切色带生硬,网格线均匀平铺,缺乏层次,"模板感"强。
2. 四张指标卡共用同一条彩虹左边条,无差异化;卡片阴影单层、偏重。
3. 时间范围是 9 个独立描边按钮,视觉噪音大。
4. 图表配色饱和度高(纯红/纯绿),tooltip 与整体风格脱节。
5. 交互反馈弱:卡片无悬停反馈、表格行无 hover 状态。

目标:保留青绿品牌色与浅色默认主题,对浅色/深色两套主题做全面视觉精修,重点重做背景。

## 设计决策

### 1. 背景(核心改动)

- 移除斜切色带(`.signal-bg`)与均匀网格平铺。
- 新背景 = 柔和底色渐变 + 3 处大尺寸低透明度"极光"径向光晕(青绿/青色/琥珀)+ 带径向蒙版的细网格(边缘淡出)+ 极低透明度噪点纹理(SVG feTurbulence,防止渐变色带)。
- 深色模式同构:深邃蓝黑底(#040810 → #0a1122),光晕换为青绿+靛蓝+微琥珀,网格与噪点透明度相应调整。

### 2. 卡片(.glass)

- 多层阴影(1px 锐利层 + 中距离柔和层 + 远距离品牌色晕染层),顶部 1px 内高光,边框更细腻。
- 悬停:轻微上浮(translateY(-2px))+ 阴影加深 + 边框着色,200ms 过渡。

### 3. 指标卡

- 去掉统一彩虹左边条,改为每卡独立强调色:总请求=青绿、成功率=翡翠绿、消耗额度=琥珀、总Tokens=天蓝。
- 强调色应用于:图标徽章底色、卡片角部径向光晕、顶部细渐变线。
- 大数字启用 tabular-nums,避免刷新时跳动。

### 4. 控件

- 时间范围改为一体式分段控件(单个胶囊容器 + 内部滑动选中块视觉)。
- 头部图标按钮统一为 `.icon-btn` 样式;RUN 徽章加呼吸点动画。
- 下拉面板、输入框圆角/边框/焦点环统一;自定义细滚动条。

### 5. 图表(ECharts)

- 成功线 #10b981(翡翠)、失败线 #f43f5e(玫红)、耗时线 #f59e0b(琥珀),总请求柱保留青绿渐变并降低顶部饱和。
- tooltip 玻璃化(圆角、阴影、模糊背景色),分割线更淡,legend 圆角图标。
- 雷达图配色对齐新调色板。

### 6. 表格

- 行 hover 背景着色 + 过渡;成功率按阈值着色(≥99% 绿 / ≥95% 琥珀 / 以下玫红)。

## 不做的事

- 不改布局结构、栅格、信息架构。
- 不改任何 JS 业务逻辑、API 调用、状态管理。
- 不引入新依赖(仍为 Tailwind CDN + ECharts + Lucide)。
- 不改浅色默认 + 深色切换的主题机制。

## 验证标准

1. `go run ./cmd/server` 演示数据模式下,浅色/深色两套主题分别截图检查:背景无生硬切边、卡片层次清晰、图表配色协调。
2. 主题切换、时间范围切换、表格展开、模型下拉、表格/图表视图切换均正常。
3. 无控制台报错。
